# Chapter 1: Introduction to Machine Learning

## 📌 What is Machine Learning?
Machine Learning is the process of extracting knowledge from data.  
It enables systems to learn patterns and make decisions or predictions without being explicitly programmed.

---

## ❓ Why Machine Learning?
In earlier systems, applications were built using hard-coded rules (if-else statements).

### Limitations of hard-coded rules:
1. The logic is domain-specific — changing the task often requires rewriting the entire code.
2. Designing rules requires deep understanding of how decisions should be made, which can be complex.

Machine Learning overcomes these issues by learning directly from data.

---

## 🧩 Problems Machine Learning Can Solve
- Tasks where rules are too complex to define manually  
- Problems involving large amounts of data  
- Situations where patterns need to be discovered automatically  
- Examples: spam detection, recommendation systems, image recognition  

---

## 🔍 Types of Machine Learning

### 1. Supervised Learning
- Learns from labeled data  
- Used for prediction tasks  
- Examples: Classification, Regression  

### 2. Unsupervised Learning
- Works on unlabeled data  
- Used to find hidden patterns  
- Examples: Clustering, Dimensionality Reduction  

---

## 📊 Understanding Data in Machine Learning
- Data is typically represented as a **table**
- Each **row** represents a data point (sample)
- Each **column** represents a feature (attribute)

⚠️ Note:  
Machine Learning models cannot make meaningful predictions if the data does not contain useful or relevant information.

---

## 🧠 Know Your Data
Understanding your data is the most important part of Machine Learning.

You should:
- Understand what the data represents  
- Know how it relates to the problem  
- Identify patterns, limitations, and biases  

---

## ❓ Questions to Ask While Building a Model
1. What problem am I trying to solve? Can my data answer it?  
2. How can I frame this as a machine learning problem?  
3. Do I have enough data to represent the problem?  
4. What features have I extracted, and are they useful?  
5. How will I measure success?  
6. How will the model integrate with the final system?
